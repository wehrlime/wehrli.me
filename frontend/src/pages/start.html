Technologies

Atwood's Law states, that "Any application that can be written in JavaScript, will eventually be written in JavaScript".


I support this hypothesis. JavaScript enables good user experience on the internet. It is accessible for everyone, runs in every browser on any computer and smartphone. This is why, in my opinion, JavaScript and the technologies that emerged from it, like TypeScript, SPA frameworks and Node.js, are the best things that has happened to software engineering and probably mankind in the last three decades.

For these reasons, my focus in software engineering lies on technologies, that emerged from JavaScript.



Front end

The front end of a web application is the parts, which is executed within a browser. Users can interact with it and it fetches data from and sends data to the back end, which runs on a server.

In the front end Angular, React, Vue.js, TypeScript, HTML and CSS are some of the technologies, I am using to build your next software project.

Back end

The back end is the part on a web application, which is executed on a server. It computes data, reads and writes to databases and is responsible to do things, that can not or should not be done in the front end.

In the back end I use TypeScript as well. Here it is used within Node.js, which is a runtime environment, that can run JavaScript like a Browser.

As a Node.js framework I use NestJS.

As Node.js headless CMS' I use Strapi or Ghost.

Mobile apps

Mobile apps can be built with multiple technologies. Because I use JavaScript, I predominantly use Ionic Framework, but React Native or NativeScript are good alternatives too.

The big advantage of so called hybrid mobile apps is, that only one app needs to be built, to support both iOS and Android.

Web server

As web server technology I use Nginx or Apache and Linux as operating system, depending on the available server.

For Node.js process management I use pm2.

For scalable back ends I use Kubernetes.

Database

I like best, to use relational database systems with SQL, because data relations can be depicted within the database structure.

I rely on ORMs like Prisma, MikroORM or TypeORM. Theses ORMs can manage a diverse range of databases, like SQLite, MySQL, MongoDB, PostgreSQL or Redis.
